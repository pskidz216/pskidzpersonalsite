The neighborhoods page.

Bond's whole brand argument lives on /pages/new-york-neighborhoods — fragrances inspired by the city's neighborhoods. Today the page runs as a directory: five stacked image cards with a "shop now" link. It's not making the case. Three section options follow, each paired to a homepage option from Page 01, each shippable as a standalone Shopify section so the merchandiser can swap between them in the theme editor.

01 / Current state

What's running on /pages/new-york-neighborhoods today.

Five full-bleed image cards stacked vertically: image · neighborhood name · "SHOP NOW" link. No editorial framing, no fragrance previews, no scent counts, no hierarchy, no mobile-specific treatment beyond default Shopify image scaling. The brand's most-defining concept gets a directory listing.

Five issues to fix
  1. 01No editorial framing. The page opens directly into tiles. No headline, no body paragraph, no statement of premise. The brand's central thesis ("a scent for every neighborhood") never gets articulated.
  2. 02No fragrance preview on the tiles. Each tile is a black box — the user can't see what fragrances live inside Uptown vs. Downtown vs. Parks. Every collection feels equally generic.
  3. 03No scent count or featured product. Eight fragrances live in Uptown; four in Beaches. The user has no way to know which neighborhoods are deep-catalog vs. light. No discovery hook.
  4. 04No hierarchy. Every tile is identical in size and treatment. Editorial design at this register privileges rhythm and asymmetry — the page reads as wallpaper, not as a magazine.
  5. 05Mobile is the desktop, smaller. No mobile-specific treatment. Five full-bleed images at 375px wide is a slow scroll with low engagement and a 2.8MB asset payload.
02 / Reference patterns

How the niche-luxury houses run their discovery / collection pages.

Bond's neighborhoods page is a hybrid — half PLP, half editorial brand-world page. Five reference houses, five different ways to handle the same problem. The pattern: editorial framing first, asymmetric hierarchy, named context (perfumer, neighborhood, story), and one path forward per tile, not "shop now" repeated five times.

Le Labo
City Exclusives
"City Exclusives" page runs a vertical mosaic of 13 city-named fragrances. Each tile: scent number, city, perfumer, year. Editorial copy sets the premise. No "shop now" — the tile IS the shop link.
Frederic Malle
Editions de Parfums
Perfumer-first — each fragrance has its own editorial spread with the perfumer's portrait and story. Discovery happens through context, not category navigation.
Byredo
Collection storytelling
Collections like "Night Veils" and "Mister Marvelous" get long-form pages — full-width imagery, alternating text/image blocks, scroll-driven story. Tiles only appear after the story has been told.
Hermès Parfums
Editorial spreads
"Les Colognes Hermès" runs as a magazine spread: large hero quote, perfumer attribution, supporting product cards in asymmetric grid. Maximum brand-world weight before any commerce signal.
Parfums de Marly
Story + map
"Les Lieux" page runs an interactive map of Paris with location-based fragrance dots. The literal geography becomes the navigation — exactly the move available to Bond if it commits.
Bond No. 9 today
5 stacked tiles · No editorial
No headline · no story · no fragrance previews · no hierarchy · no map · no perfumer attribution. Closer to a generic Shopify collection grid than to any of the niche-luxury references Bond cites.
03 / Option A
Editorial Mosaic · asymmetric grid · ships fastest

The magazine-spread tile mosaic.

Pairs with Homepage B · Two Doors

Treat the five neighborhoods like a magazine cover spread. One large hero tile (Uptown or Downtown — settable in schema) and four supporting tiles arranged asymmetrically around it. Each tile is a portal: image, neighborhood name in serif, one-line tagline. Hover reveals scent count + featured fragrance. Above the grid, an editorial intro band — large serif headline + 2–3 sentence body — fixes the missing voice. Lowest build complexity of the three. Pairs with Homepage B because both options optimize for fast comprehension and direct conversion paths.

Why this works for the neighborhoods page

  • Lowest build cost — single Liquid section, standard CSS grid, no custom artwork
  • Asymmetric hierarchy gives the page rhythm — every tile no longer screams equally
  • Hover reveal carries the missing data (scent count, featured fragrance) without cluttering the default state
  • Editorial intro band fixes the voice problem in 24 lines of HTML
  • Reflows cleanly to single column on mobile — no custom mobile treatment needed beyond minor type scaling
  • Hero tile is settable per-block — the merchandiser can rotate Uptown / Downtown / Parks based on season or campaign

Trade-offs

  • Hover-reveal data is invisible to keyboard users without focus styles — needs a tap-and-hold equivalent on touch and a focus state on tab
  • Doesn't capitalize on Bond's geography-as-fragrance thesis — could be a mosaic for any product, not specifically for neighborhoods
  • Needs five quality images cropped to mosaic ratios — current CDN files are landscape, hero tile wants portrait or square
  • Less differentiated from competitor brand pages than Option B
04 / Option B
Interactive NYC Map · stylized SVG · highest brand differentiation

The map of New York, made literal.

Pairs with Homepage C · The Hybrid

A stylized SVG illustration of the relevant NYC geography — Manhattan core plus a hint of Brooklyn, Queens, and the Long Island beaches. Five labeled hotspots, one per neighborhood. Hover or focus a hotspot: it pulses, a side panel slides in from the right showing image, name, tagline, featured fragrance card with bottle and price, plus two CTAs. The whole pitch of Bond No. 9 is geography-as-fragrance — making the geography literal and explorable on the brand page is a defensible differentiator no competitor has. Pairs with Homepage C because both options earn their weight by leaning into the brand's most distinctive idea.

Why this works for the neighborhoods page

  • Most editorial-feeling option — closest to Byredo, Hermès Parfums, Frederic Malle long-form pages
  • Story paragraph slot fixes the voice problem most thoroughly — each neighborhood gets ~50 words of brand-world prose

Trade-offs

Mobile principles · apply to all three

  • — No image asset over 200KB on mobile. Use Shopify's image_url filter with width: 750 for mobile-served srcset entry. The current page ships 2.6MB of imagery — that's killing LCP.
  • — Lazy-load everything below the fold. First two tiles / first map state / first story section eager; everything else loading="lazy".
  • — No layout shift. Every image gets explicit width/height or aspect-ratio CSS. CLS target: 0.
  • — Touch targets 44px minimum. Map hotspots, chip rail items, "Explore" CTAs all ≥44px hit area. WCAG 2.1 AA.
  • — Respect prefers-reduced-motion. Parallax, hotspot pulse, hover-reveal transitions all freeze when set.
07 / Pairing summary

Which neighborhoods option goes with which homepage.

Page-to-page register has to agree. A cinematic editorial homepage with a stripped-down conversion-grid neighborhoods page reads inconsistent. A two-doors funnel homepage with a slow-scroll five-spread story reads inconsistent. The pairings below are based on register match, not letter match — the option labels (A · B · C) follow the spec naming, but the homepage pairings cross over deliberately.

Option Pair Why this pair
A Editorial Mosaic → Homepage B · Two Doors Both options optimize for fast comprehension and direct conversion paths. Two Doors splits the homepage into two clear funnels at the hero; the mosaic gives the neighborhoods page the same logic — five tiles, one path each, hover reveals the data. Lowest joint build cost. The pragmatic ship-fast pairing.
B Interactive Map → Homepage C · Hybrid Both options earn their weight by leaning into Bond's most distinctive idea. Homepage C is the "best of both" — cinematic hero + Two Doors at scroll. Map is the same move at the page level — it commits to geography-as-fragrance as the brand's literal navigation. The brand-defining pairing.
C Scrolling Stories → Homepage A · Editorial Front Both options bet on long-form brand-world reading time over fast conversion. Editorial Front gives the homepage a single cinematic frame with restrained scroll; Stories gives the neighborhoods page five magazine spreads with parallax and editorial copy. The luxury-magazine pairing.

Cross-cutting decisions · apply to all three

  • — Editorial intro band, every option. Eyebrow + headline + 2–3 sentence body before any tiles, map, or stories. The voice gap is the page's biggest current failure — fix it everywhere.
  • — Scent count per neighborhood, every option. Pulled live from the linked collection (collection.products_count). Never hardcoded. The user should always know how deep the collection runs before they click in.
  • — Featured fragrance per neighborhood, every option. One product card minimum — bottle, name, price (Shopify money filter), 1-line note family. If featured_product is empty, fall back to first product in collection.
  • — Two link types per neighborhood, every option. The site uses /pages/neighborhood/{slug} for editorial and /collections/{handle} for product list. Schema must expose both — merchandiser picks per block.
  • — Reuse existing CDN imagery. Five files already on the Bond CDN (Uptown · Downtown · NY Parks · City Scents · Beaches). Don't re-shoot, don't re-upload. Schema fields use image_picker for swappability.
  • — Color schemes only, never hardcoded hex. Sections respect the merchandiser's chosen color_scheme setting. Light + dark scheme variants must both pass WCAG 2.1 AA contrast.
08 / Shopify build path

How this ships as three Shopify sections.

All three options are built as standalone sections in the existing Online Store 2.0 theme. The merchandiser swaps between them in the theme editor — no Liquid edits required. Default state ships Mosaic active, Map and Stories disabled but present so the team can preview by toggling.

Deliverable file structure

  • sections/neighborhoods-mosaic.liquid — Option A · Editorial Mosaic. Self-contained Liquid + scoped {% stylesheet %} + scoped {% javascript %}. Hover-reveal logic in vanilla JS (no theme JS dependency).
  • sections/neighborhoods-map.liquid — Option B · Interactive Map. Inline SVG (no referenced asset, ≤15KB), <dialog>-based side panel with focus trap, x_percent/y_percent range fields per hotspot block.
  • sections/neighborhoods-stories.liquid — Option C · Scrolling Stories. IntersectionObserver + requestAnimationFrame parallax (never background-attachment:fixed), soft scroll-snap, alternating left/right grid logic by block index.
  • templates/page.new-york-neighborhoods.json — Updated to include all three sections, Mosaic active by default, Map and Stories "disabled": true. Merchandiser previews by toggling.
  • README.md — How each section is structured, how to swap which one is live, schema field map, how to add/remove a neighborhood block.

Acceptance criteria · all three options

  • — Lighthouse mobile ≥90 · CLS 0 · LCP <2.5s on Fast 3G simulation
  • — Keyboard reachable · every interactive element via Tab in logical order, focus rings visible, no traps
  • — Screen reader sensible · heading hierarchy, alt text, descriptive CTAs (no "click here")
  • — prefers-reduced-motion: reduce respected · parallax, hover scales, hotspot pulses freeze
  • — Works at 320px wide without horizontal scroll · iOS Safari 15+, last 2 versions Chrome/Safari/Firefox/Edge
  • — Light + dark color schemes · both pass WCAG 2.1 AA contrast without manual overrides
